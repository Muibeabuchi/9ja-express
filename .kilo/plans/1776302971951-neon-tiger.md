# Plan: Separate Auth Layout without Navbar

## Goal
Sign-in and sign-up pages should not share the main Navbar/Footer layout. They need their own minimal layout.

## Approach
Use TanStack Router's nested route layout pattern:

1. Create a new `auth.tsx` layout route file at `src/routes/auth.tsx`
   - This layout will have NO Navbar and NO Footer
   - Just a minimal container with Outlet and Toaster
   - This serves as the parent layout for `/sign-in` and `/sign-up`

2. Modify the route definitions for sign-in and sign-up
   - Make them child routes under the auth layout
   - Use the `createChildRoute` or nest them under the auth route

## Implementation Steps

### Step 1: Create auth layout route
Create `src/routes/auth.tsx`:
```typescript
import { createFileRoute, Outlet } from "@tanstack/react-router"
import { Toaster } from "@/components/ui/sonner"

export const Route = createFileRoute("/auth")({
  component: () => (
    <>
      <div className="flex min-h-screen items-center justify-center bg-background">
        <Outlet />
      </div>
      <Toaster />
    </>
  ),
})
```

### Step 2: Move sign-in and sign-up under auth layout
The file-based routing in TanStack Router uses directory structure. Two options:

**Option A:** Create `/auth` directory and put sign-in.tsx, sign-up.tsx inside it
- `src/routes/auth/sign-in.tsx` → maps to `/auth/sign-in`
- `src/routes/auth/sign-up.tsx` → maps to `/auth/sign-up`
- These will automatically be children of `/auth` route

**Option B:** Keep files in place but add parent route config (less clean)

Recommend Option A for cleaner routing.

### Step 3: Delete the old route files
Delete:
- `src/routes/sign-in.tsx`
- `src/routes/sign-up.tsx`

### Step 4: Recreate in auth directory
Create new files:
- `src/routes/auth/sign-in.tsx`
- `src/routes/auth/sign-up.tsx`

These will automatically be nested under `/auth` layout.

## Additional: Auth Guard in Auth Layout
Add a `beforeLoad` hook to the auth layout (`src/routes/auth.tsx`) to redirect authenticated users away from auth pages:

```typescript
beforeLoad: () => {
  const session = sessionStorage.getItem("pmt_session")
  if (session) {
    throw redirect({ to: "/" })
  }
}
```

This guards ALL auth pages at once (sign-in and sign-up), which is cleaner than having it in each individual page.

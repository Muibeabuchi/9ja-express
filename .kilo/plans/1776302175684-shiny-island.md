# Mock Authentication Implementation Plan

## Overview
Implement mock authentication for the "Book a Trip" and "Hire a Bus" flows. Users must sign in/sign up before proceeding to checkout. After authentication, redirect back to the intended destination.

## Architecture

### Auth Storage
- **LocalStorage**: Store registered users (persists browser close)
  - `pmt_users`: Array of registered users `{ id, email, password, name, createdAt }`
- **sessionStorage**: Store current session (only survives page refresh)
  - `pmt_session`: Current logged-in user `{ id, email, name }` or `null`

### Redirect Strategy
- Store intended destination in sessionStorage under key `pmt_auth_redirect`
- After login/signup, read and clear this value to redirect user back

### Route Guarding Strategy
Using **client-side guards** at navigation points (simpler for demo):

1. **Seat Selection** → `handleConfirm()` checks auth before `navigate('/checkout')`
2. **Hire Fleet** → Proceed button in sheet/drawer checks auth before `navigate('/hire-checkout')`
3. **Hire Checkout** → `beforeLoad` hook in route checks auth and redirects to `/sign-in` if not authenticated

This approach provides protection at the exact points where users try to access protected flows.

## Implementation Steps

### Step 1: Create Auth Store with Zustand
Create `src/stores/auth-store.ts`:
- Use Zustand for global state management
- State: `user` object or null, `isInitialized` boolean
- Actions:
  - `login(email, password)` - validates against localStorage, sets session in sessionStorage
  - `signup(name, email, password)` - saves to localStorage, auto-logs in
  - `logout()` - clears session
  - `initialize()` - restores session from sessionStorage on app load
- Sync with sessionStorage for persistence (survives page refresh, not browser close)
- Export hook: `useAuthStore()` for components to access auth state

### Step 2: Create Sign-In Page
Create `src/routes/sign-in.tsx`:
- Route: `/sign-in`
- Fields: email, password
- Use TanStack Form + Zod validation
- On success: set session, redirect to stored URL or home
- Link to sign-up page

### Step 3: Create Sign-Up Page
Create `src/routes/sign-up.tsx`:
- Route: `/sign-up`
- Fields: name, email, password, confirm password
- Use TanStack Form + Zod validation
- On success: set session, redirect to stored URL or home
- Link to sign-in page

### Step 4: Update Navbar User Menu
Modify `src/components/navbar.tsx`:
- Replace "Account" button with Avatar dropdown menu
- When logged out: show "Account" that links to sign-in
- When logged in: show avatar with dropdown containing:
  - User name/email
  - "Log Out" button
- Use shadcn/ui: Avatar, DropdownMenu

### Step 5: Add Auth Guard to Flows

**Seat Selection Flow:**
In `seat-selection-page.tsx`, modify `handleConfirm`:
```typescript
const handleConfirm = () => {
  if (!isAuthenticated) {
    sessionStorage.setItem('pmt_auth_redirect', JSON.stringify({
      to: '/checkout',
      search: { busId, seatNumbers: selectedSeats, departureDate }
    }))
    navigate({ to: '/sign-in' })
    return
  }
  navigate({ to: '/checkout', search: { ... } })
}
```

**Hire Fleet Flow:**
Find the proceed button in hire-fleet-page and add auth check before navigation to `/hire-checkout`.

**Hire Checkout Route:**
In `src/routes/hire-checkout.tsx`, add a `beforeLoad` hook:
```typescript
beforeLoad: () => {
  const session = sessionStorage.getItem('pmt_session')
  if (!session) {
    sessionStorage.setItem('pmt_auth_redirect', JSON.stringify({
      // preserve current search params
    }))
    throw redirect({ to: '/sign-in' })
  }
}
```

## Validation Schemas

**Sign-In:**
```zod
z.object({
  email: z.string().email("Invalid email"),
  password: z.string().min(1, "Password required")
})
```

**Sign-Up:**
```zod
z.object({
  name: z.string().min(2, "Name must be at least 2 characters"),
  email: z.string().email("Invalid email"),
  password: z.string().min(6, "Password must be at least 6 characters"),
  confirmPassword: z.string()
}).refine(data => data.password === data.confirmPassword, {
  message: "Passwords don't match",
  path: ["confirmPassword"]
})
```

## Acceptance Criteria

1. User cannot navigate to `/checkout` without being authenticated
2. User cannot navigate to `/hire-checkout` without being authenticated
3. When unauthenticated user clicks "Proceed to Payment" in seat selection, they're redirected to `/sign-in`
4. When unauthenticated user clicks "Proceed" in hire-fleet, they're redirected to `/sign-in`
5. After sign-in/sign-up, user is redirected back to the intended checkout page with all params preserved
6. Navbar shows user avatar with dropdown when logged in
7. User can log out from the avatar dropdown
8. Auth session persists only across page refreshes (cleared on browser close)

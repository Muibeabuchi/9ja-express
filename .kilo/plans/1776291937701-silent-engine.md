# Plan: Convert mockData to JSON with Zod Validation

## Goal

Convert the mockData from TypeScript to JSON file and add Zod validation for type safety, fixing the "union type too complex" error.

## Files to Modify

### 1. `src/data/mockData.json` (new file)

- Export the generated buses data as JSON
- Keep locations, bookings, globalSurcharges, charterFleet as-is

### 2. `src/data/mockData.ts` (update)

- Import data from mockData.json
- Apply Zod schema validation

### 3. `src/data/schemas.ts` (new file)

- Create Zod schemas for all data types:
  - Location enum
  - BusType enum
  - Bus Schema
  - Booking Schema
  - CharterVehicle Schema
  - GlobalSurcharges Schema
  - AppData Schema

## Implementation Steps

1. **Create `src/data/mockData.json`**
   - Extract the `buses` array from current mockData.ts
   - Add all other data (locations, bookings, globalSurcharges, charterFleet)
   - Save as valid JSON

2. **Create `src/data/schemas.ts`**
   - Define Zod schemas:

   ```typescript
   import { z } from "zod"

   export const LocationEnum = z.enum([...18 locations])

   export const BusTypeEnum = z.enum(["Executive Coach", "Smart Coach", "Blazer"])

   export const BusSchema = z.object({
     id: z.string(),
     name: z.string(),
     type: BusTypeEnum,
     totalSeats: z.number(),
     price: z.number(),
     departureTime: z.string(),
     arrivalTime: z.string(),
     from: LocationEnum,
     to: LocationEnum,
     availableSeats: z.array(z.number()),
     rating: z.number(),
     duration: z.string(),
   })

   export const AppDataSchema = z.object({
     locations: z.array(LocationEnum),
     buses: z.array(BusSchema),
     bookings: z.array(BookingSchema),
     charterFleet: z.array(CharterVehicleSchema).optional(),
     globalSurcharges: z.object({...}).optional(),
   })
   ```

3. **Update `src/data/mockData.ts`**
   - Import JSON data
   - Validate with Zod and export validated data:

   ```typescript
   import rawData from "./mockData.json"
   import { AppDataSchema } from "./schemas"

   const validatedData = AppDataSchema.parse(rawData)
   export const data = validatedData
   ```

4. **Update imports across the app**
   - Where `data` is imported, update paths if needed

## Benefits

- Fixes "union type too complex" TypeScript error
- Runtime type validation with Zod
- JSON loads lazily, avoiding eager type inference
- Maintains full type safety after validation

## Notes

- `resolveJsonModule` already enabled in tsconfig.json
- May need to install Zod if not present: `npm install zod`

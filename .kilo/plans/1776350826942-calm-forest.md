# Plan: User Bookings Dashboard

## Goal

Create a protected page where authenticated users can view all their booked trips and hired buses. Each booking/hire should display its status based on whether it has expired (past the travel/hire date).

---

## 1. Data Model Updates

### Challenge

Existing bookings in localStorage (`pmt_bookings` and `pmt_hire_bookings`) don't store a `userId`. They only store `fullName`, `email`, and `phone`.

### Solution

- **For new bookings**: Add `userId` field to both `StoredBooking` and `StoredHireBooking` interfaces
- **For existing bookings**: Match by logged-in user's email

### Files to modify:

1. `src/lib/bookingStorage.ts` - Add `userId?: string` to `StoredBooking`
2. `src/lib/hireBookingStorage.ts` - Add `userId?: string` to `StoredHireBooking`

### New functions needed:

- `getBookingsByUserId(email: string)` - Get all bus bookings by user email
- `getHireBookingsByUserId(email: string)` - Get all hire bookings by user email

---

## 2. Storage Updates

### Modify checkout pages to save userId

When a user completes a booking, save the current user's ID:

- `src/routes/checkout.tsx` (or `src/components/pages/checkout-page.tsx`)
- `src/routes/hire-checkout.tsx` (or `src/components/pages/hire-checkout-page.tsx`)

## 4. New Page Component

### Create `src/components/pages/my-bookings-page.tsx`

**Features:**

1. **Authentication check** - Redirect to sign-in if not logged in
2. **Fetch user's bookings** - Load both bus and hire bookings filtered by user email
3. **Tabs or sections** - Separate views for "Trips" (bus bookings) and "Charters" (hire bookings)
4. **Expiry status** - Compare `departureDate` (bus) or `startDate` (hire) with current date:
   - If date < today → **Expired** (show grayed out/strikethrough styling)
   - If date >= today → **Active** (show normal styling)
5. **Empty state** - Show friendly message if no bookings found

**UI Design:**

- Use existing card components from the project
- Display: booking ref, route (from → to), date, status badge (Active/Expired)
- Link each card to `/manage-booking?ref=XXX` for details

## 7. Expiry Logic

```typescript
const isExpired = (dateStr: string) => {
  const travelDate = new Date(dateStr)
  const today = new Date()
  today.setHours(0, 0, 0, 0)
  return travelDate < today
}
```

Apply different styling based on this:

- Active: Normal colors, full opacity
- Expired: Reduced opacity (50%), gray overlay, "Expired" badge

# Search Results Page - Filtering & Sorting Implementation Plan

## Current State
The `search-results-page.tsx` has mock UI filters that aren't connected to any functionality:
- Bus Type filter (shows "AC", "Non-AC", "Sleeper" - not matching data)
- Price Range slider (static "₦5k - ₦50k")
- Departure Time checkboxes (non-functional)
- Sort dropdown (non-functional)

## Mock Data Analysis

### Available Bus Types
The mock data has 3 distinct bus types:
- `Executive Coach` (e.g., Stellar Liners, God is Good Motors)
- `Smart Coach` (e.g., BlueJet Transit, Peace Mass Transit)
- `Blazer` (e.g., Star Eleven, Road Master)

### Available Price Range
- Min: ₦10,000 (bus id 41)
- Max: ₦112,000 (bus id 3)

### Departure Time Distribution
- Early Morning: 05:00 - 12:00
- Afternoon: 12:00 - 17:00  
- Night: 17:00 - 05:00

## Required shadcn/ui Components

Install these components before implementation:

```bash
npx shadcn@latest add dropdown-menu slider checkbox
```

Components needed:
1. **dropdown-menu** - for the sorting dropdown (replaces the non-functional sort UI)
2. **slider** - for the price range filter (replaces HTML range input)
3. **checkbox** - for departure time filter (replaces custom checkbox UI)

---

## Implementation Tasks

### 1. Add Filter State Management
Add state for:
- Selected bus types (array)
- Price range (min/max)
- Selected departure time periods (array)
- Sort option (string)

### 2. Implement Dynamic Price Range
- Calculate min/max prices from filtered buses dynamically
- Update range slider bounds based on current data

### 3. Update Bus Type Filter
- Change buttons to match actual data: "Executive Coach", "Smart Coach", "Blazer"
- Make them toggleable with active state styling

### 4. Make Departure Time Filter Functional
- Use shadcn checkbox components for time periods: Early Morning, Afternoon, Night

### 5. Implement Sorting Dropdown
- Use shadcn dropdown-menu for sorting options:
  - **Earliest First** (default) - sort by departureTime ascending
  - **Price: Lowest to Highest** - sort by price ascending
  - **Price: Highest to Lowest** - sort by price descending

### 6. Apply Filters & Sorting
- Chain filtering logic before displaying results
- Update `filteredBuses` useMemo to include all filters + sorting

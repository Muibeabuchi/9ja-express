# The Prestige Horizon Plan

## Goal
- Refactor the homepage into a cohesive cinematic experience that extends the current hero/navbar visual language across the full landing page.
- Eliminate horizontal overflow/layout shift at the root and section levels.
- Extract the active hero design DNA into reusable global tokens/utilities for Tailwind v4 CSS-first theming.
- Replace the shared footer globally with the new premium footer variant.

## Findings

### Current structure
- `src/routes/__root.tsx` owns the global shell with `Navbar`, `Outlet`, and shared `Footer`.
- `src/components/pages/home-page.tsx` is currently monolithic and mixes hero, destinations, and legacy feature sections inline.
- Tailwind v4 styling is centralized in `src/index.css`; there is no `tailwind.config.*` file in the repo.
- The visual system already leans on `Inter` + `Manrope`, rounded premium surfaces, translucent overlays, and `motion/react` entry animations.

### Likely horizontal overflow causes
- `src/components/pages/home-page.tsx:21` uses `overflow-visible` on the hero wrapper while the hero background image is animated to `scale: 1.05` in `src/components/pages/home-page.tsx:22`, which can push pixels outside the viewport.
- `src/components/search-widget.tsx:162` uses `overflow-x-auto` around the tabs block, and `src/components/ui/tabs.tsx:26` uses `inline-flex w-fit`, which can create a horizontal scrollbar or visible page shift on smaller screens.
- `src/routes/__root.tsx:13` lacks a global overflow guard on the app shell.
- Some cards use hover scale (`src/components/destination-card.tsx:16`) and several large-width, no-wrap controls in `src/components/search-widget.tsx`, so section wrappers need clipping and `max-w-full` discipline.

### Existing assets/patterns to reuse
- `src/index.css:313` already defines typography tokens and semantic color slots.
- `src/index.css:394` defines a `.glass` utility, but it does not match the requested hero glass treatment yet.
- `src/components/hire-fleet/fleet-catalog.tsx` contains an established fleet card language suitable for the new fleet preview.
- `src/data/mockData.ts:228` contains charter fleet data for Sienna, Hiace, coaches, and related amenities.
- Motion patterns in `src/components/pages/home-page.tsx` and `src/components/navbar.tsx` already use premium fade/rise easing that can be standardized for section reveals.

## Confirmed decisions
- Keep the current live typography pairing as the system standard: `Inter` + `Manrope`.
- Replace the shared footer globally rather than rendering a homepage-only footer.

## Implementation plan

### 1. Stabilize layout and remove horizontal shift
- Update the global app shell in `src/routes/__root.tsx` to enforce `max-w-full`/`overflow-x-hidden` on the top-level wrapper and `main` region.
- Update base layout styles in `src/index.css` to ensure `html`, `body`, and the app root do not permit horizontal overflow.
- Refactor the hero wrapper in `src/components/pages/home-page.tsx` to use `overflow-hidden`, `max-w-full`, and width-safe containers.
- Audit homepage sections and new components to replace any `w-screen` usage with `w-full` and add `max-w-full` to image wrappers, motion wrappers, and grids.
- Adjust `src/components/search-widget.tsx` and, if necessary, `src/components/ui/tabs.tsx` so the hero tab row remains mobile-safe without causing document-level overflow.

### 2. Extract and align the design system in Tailwind v4 CSS
- Extend `src/index.css` with hero-derived tokens for:
  - Deep Midnight background
  - Golden Hour accent
  - Glass surface treatment
  - Soft Elevation shadow
- Update the existing `.glass` utility to match the requested style: `bg-white/5 backdrop-blur-md border border-white/10`.
- Add supporting utilities/tokens for premium section surfaces, accent glows, and reusable section spacing so the new homepage does not rely on repeated raw values.
- Preserve semantic naming so the homepage can use tokens while other routes continue to function.

### 3. Modularize the homepage and preserve legacy content in comments
- In `src/components/pages/home-page.tsx`, comment out the existing non-hero homepage sections rather than deleting them.
- Convert `home-page.tsx` into a clean composition file that renders:
  - existing hero/search experience
  - `FeaturesGrid`
  - `PopularRoutes`
  - `ProcessFlow`
  - `FleetPreview`
  - any bridging CTA/details needed for pacing
- Keep the hero visual language intact while bringing its spacing/container rules in line with the new global system.

### 4. Build the new homepage section components
- Create new modular homepage components, likely under `src/components/home/` or `src/components/pages/home/`, for:
  - `FeaturesGrid.tsx`
  - `PopularRoutes.tsx`
  - `ProcessFlow.tsx`
  - `FleetPreview.tsx`
  - `EnhancedFooter.tsx` logic folded into the shared footer implementation or extracted into footer subcomponents
- Reuse a shared section shell pattern (`mx-auto max-w-7xl px-4 sm:px-6`) to keep spacing and widths consistent.
- Ensure every section is responsive, airy, and visually connected to the hero through background layering, typography, and glass surfaces.

### 5. Section-specific design and content direction

#### FeaturesGrid
- Build a bento-style grid with four feature cards for real-time tracking, professional drivers, climate control, and seamless refunds.
- Use Lucide icons with subtle accent glow and soft-elevation shadows.
- Mix card sizes and negative space to avoid repetitive equal-column layouts.

#### PopularRoutes
- Create premium destination cards for Lagos, Abuja, and Port Harcourt with large-format imagery.
- Add hover image scale and a sliding price badge (`Starting from ₦X,XXX`).
- Replace the legacy `DestinationCard` usage on the homepage if the new design requires richer interactions and tighter overflow control.

#### ProcessFlow
- Implement a horizontal three-step explainer: Search -> Select -> Go.
- Add an animated tracking line/bus-route motif that remains subtle on desktop and stacks cleanly on mobile.
- Use clean typography and restrained iconography so the section acts as a visual breath between denser content blocks.

#### FleetPreview
- Source content from `src/data/mockData.ts` and select the most relevant fleet options to represent Sienna, Hiace, and a premium coach/jet-equivalent highlight.
- Reuse the `src/components/hire-fleet/fleet-catalog.tsx` card language as the visual baseline, but simplify it into a homepage showcase focused on comfort specs such as legroom, AC, and WiFi.
- Decide during implementation whether this is a snap carousel or featured-card layout based on the smallest, most stable solution for responsiveness and performance.

#### Global footer replacement
- Replace `src/components/footer.tsx` with a 4-column premium footer using deep charcoal/deep midnight surfaces.
- Include columns for Company, Services, Support, and Legal.
- Add minimalist social icons and a glassmorphism newsletter signup titled `Join the Journey`.
- Keep the footer shared via `src/routes/__root.tsx` so all routes inherit the redesign.

### 6. Motion system
- Apply scroll-triggered reveal animations to each homepage section using `motion` with `viewport={{ once: true }}`.
- Standardize on a fade-and-rise transition from `y: 50` to `y: 0` with the existing premium easing curve.
- Use staggered children for feature cards, route cards, process steps, and fleet items so motion feels sequenced rather than simultaneous.
- Ensure hover animations do not reintroduce overflow by clipping image wrappers and scaling internal media instead of outer layout containers where possible.

### 7. Validation and QA during implementation
- Run a focused visual audit of the homepage in mobile and desktop breakpoints after the refactor.
- Verify there is no horizontal scrollbar in the landing page shell.
- Confirm the global footer still behaves correctly on non-home routes.
- Run the project’s relevant checks/build after implementation to catch Tailwind/class/runtime issues.

# Plan: Integrate Bus Hire Reference Tracking into Search Results Page

## Goal
Modify `src/components/pages/search-results-page.tsx` to support tracking bus hire bookings by reference number, in addition to the existing bus seat search functionality.

## Current State Analysis
- The page currently displays bus search results filtered by `from`, `to`, and `departureDate` parameters.
- Hire bookings are stored in localStorage with references (format: "CHT-XXXXXXXX").
- Hire tracking is handled separately via `/manage-booking` route.
- No integration between bus search and hire tracking exists.

## Proposed Changes

### 1. Route Modification
- Update the search-results route to accept an optional `ref` parameter.
- This allows URLs like `/search-results?ref=CHT-A1B2C3D4` for hire tracking.

### 2. Component Logic Updates
- Extract `ref` from route search params alongside existing `from`, `to`, `departureDate`.
- Add conditional rendering:
  - If `ref` is provided and valid: Display hire booking details
  - If `ref` is provided but invalid: Show error message
  - If no `ref`: Show existing bus search functionality

### 3. Hire Tracking Functionality
- Import hire booking storage functions (`getHireBookingByRef`, `isValidHireBookingRef`).
- When `ref` is provided, validate and fetch booking data.
- Display hire booking details including:
  - Booking reference
  - Vehicle details
  - Passenger information
  - Dates and pricing
  - Status/confirmation details

### 4. UI Modifications
- Add a toggle or tab system to switch between "Search Buses" and "Track Hire".
- For hire tracking mode: Show a reference input field.
- Reuse existing styling patterns from the page.
- Handle empty states and error cases gracefully.

### 5. Navigation Integration
- Update navigation logic to handle hire tracking flows.
- Ensure seamless transition between search and tracking modes.

## Implementation Steps
1. Update route definition to include optional `ref` parameter.
2. Modify component to extract and handle `ref` parameter.
3. Add hire booking data fetching and validation.
4. Implement conditional UI rendering for bus search vs hire tracking.
5. Add hire booking display component/logic.
6. Test integration and error handling.
7. Update any related routing or navigation code.

## Potential Challenges
- Ensuring consistent UI/UX between bus search and hire tracking modes.
- Handling invalid or non-existent references.
- Maintaining performance with localStorage operations.
- Ensuring mobile responsiveness for new UI elements.

## Dependencies
- `hireBookingStorage.ts` functions for data access.
- Existing hire booking data structure.
- Route configuration updates.
